# 🤝 מדריך תרומה לפרויקט

תודה על העניין שלך לתרום לפרויקט PluginOtzaria! מדריך זה יעזור לך להבין את תהליך התרומה.

## קוד התנהגות

אנו מטבח קהילה טובה וכולל. כל התורמים צריכים:
- להיות מכבדים וטבוע לרעה
- קבל ביקורת בונה
- להתמקד בתוצאות, לא בפרטים אישיים

## התחלה

### דרישות קדם

- 🔗 [חשבון GitHub](https://github.com/signup)
- 💻 [Git](https://git-scm.com/) מותקן
- 🖥️ עורך קוד (VS Code מומלץ)
- 📦 Node.js (אופציונלי, רק אם צריך)

### Setup ראשוני

```bash
# 1. Fork את הריפו
# גוש לריפו וקליק "Fork" בפינה השמאלית העליונה

# 2. Clone את ה-Fork שלך
git clone https://github.com/YOUR_USERNAME/PluginOtzaria.git
cd PluginOtzaria

# 3. הוסף את הריפו המקורי כ-upstream
git remote add upstream https://github.com/lodbig/PluginOtzaria.git
```

## סוגי תרומה

### 1. 🐛 דיווח על Bug

**בדוק קודם:**
- גרסת אוצריא שלך עדכנית?
- תוכל להשחזור את הבעיה בגרסה האחרונה?
- יש Issue דומה כבר?

**בעת דיווח:**

```markdown
## תיאור
[תיאור קצר של הבעיה]

## שלבי השחזור
1. ...
2. ...
3. ...

## התנהגות צפויה
[מה צריך להיות]

## התנהגות בפועל
[מה קורה באמת]

## סביבה
- גרסת אוצריא: X.X.X
- דפדפן: Chrome/Firefox/Safari
- מערכת הפעלה: Windows/Mac/Linux

## צילומי מסך
[אם רלוונטי]
```

### 2. 💡 הצעת תכונה

**בדוק קודם:**
- האם הם כבר מתוכננים?
- האם זה מתאים לפרויקט?
- האם יש ערך לרוב המשתמשים?

**בעת הצעה:**

```markdown
## תיאור
[מה הבעיה שאתה פותר?]

## פתרון מוצע
[איך התכונה תחזיק?]

## דוגמה
[שימוש לדוגמה]

## תחלופה
[יש חלופות?]

## הערות נוספות
[כל דבר אחר]
```

### 3. 🔧 Pull Request (קוד)

**תהליך:**

```bash
# 1. עדכן את Branch הראשי
git fetch upstream
git checkout main
git merge upstream/main

# 2. צור Branch חדש
git checkout -b fix/הסבר-בקצרה

# 3. בצע את השינויים שלך
# 4. Commit לשינויים
git add .
git commit -m "🐛 Fix: תיאור התיקון"

# 5. Push לה-Fork שלך
git push origin fix/הסבר-בקצרה

# 6. פתח Pull Request בגיטהאב
```

**שמות Branch:**
- `fix/` - לתיקוני באגים
- `feature/` - לתכונות חדשות
- `docs/` - לעדכוני דוקומנטציה
- `refactor/` - לשיפור קוד

**כללי Commit:**

```bash
# טוב
git commit -m "🎨 Improve: ממשק משתמש חדש לחלון הגדרות"
git commit -m "🐛 Fix: שגיאה בשמירת נתונים במעשר"

# רע
git commit -m "update"
git commit -m "תיקון מהיר"
```

**סמלים ל-Commits:**
- 🎨 - שינויים בעיצוב
- 🐛 - תיקון בג
- ✨ - תכונה חדשה
- 📚 - דוקומנטציה
- ⚡ - שיפור ביצועים
- 🔒 - תיקון בטיחות
- 🧹 - קוד ניקיון
- ✅ - הוסף טסטים

### 4. 📚 דוקומנטציה

תרומה לדוקומנטציה חשובה מאוד!

```bash
git checkout -b docs/תיאור
# ערוך קבצי README או .md
git commit -m "📚 Docs: תיאור השינוי"
```

## טיפים לתרומה מצליחה

### ✅ בעיות שיוצרות Pull Requests מדי מהר
- ✓ בדוק את תבנית הקוד הקיימת
- ✓ שים לב לעברית בכל הצדדים
- ✓ ודא קוד נקי וקריא
- ✓ הערות בעברית או בעברית

### ✅ לפני ה-Push
```bash
# 1. בדוק את הקוד
git diff upstream/main

# 2. ודא שאין קונפליקטים
git merge upstream/main

# 3. התצפה בעיניים שוב
code .
```

### ✅ בעת פתיחת PR
- כתוב כותרת ברורה
- הוסף תיאור מפורט
- צרף דוגמה אם צריך
- תייג Issues שקשורות

**דוגמה PR תיאור:**
```
## 📝 תיאור
תיקון בעיה של אי שמירה של נתונים בתוסף מעשר כספים

## 🔗 קשור ל-Issue
Closes #42

## 🧪 בדיקה
בדקתי ב:
- Chrome 120
- Firefox 121
- Safari 17

## 📷 צילומי מסך
[צירוף צילום אם צריך]
```
