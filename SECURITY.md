# 🔒 מדיניות אבטחה

## דיווח על בעיות אבטחה

אם גילית חולשה בתוסף, **אנא אל תפתח Issue ציבורי**. במקום זאת:

### דיווור פרטי
1. לך ל: **GitHub Settings > Security > Report a vulnerability**
2. תאר את החולשה בפירוט
3. ספק דוגמה לשחזור אם אפשרי

**או** שלח דוא"ל ל:
- lodbig (בדוא"ל מאת GitHub)

### מה לצפות
- ✓ תשובה בתוך 48 שעות
- ✓ דיוק בשמירות אבטחה
- ✓ קבלת זיכוי (אם רוצה)

## עקרונות אבטחה

כל התוספים עוקבים אחר:

### 🔐 פרטיות נתונים
- ✓ כל הנתונים אחסנים בהתקן בלבד
- ✓ אין שידור לשרתים חיצוניים
- ✓ אין ביצוע כרייה נתונים

### 🛡️ טיפול נתונים רגישים
- ✓ PIN מוצפנים בהתקן
- ✓ חומש וקבלות לא משודרות
- ✓ היסטוריה מוטבעת בלבד

### 🔑 אמצעי אבטחה
- ✓ הצפנה של נתונים בהתקן
- ✓ הגנה עם קוד גישה (אופציונלי)
- ✓ טיהור נתונים כשמוחקים

## עדכוני אבטחה

התוספים יקבלו עדכוני אבטחה:
- **קריטיים**: בתוך 24-48 שעות
- **חמורים**: בתוך שבוע
- **נורמליים**: בעדכון הבא

## בדיקות אבטחה

אנו מבצעים:
- ✓ סקירת קוד ידנית
- ✓ בדיקות יד
- ✓ בדיקות תאימות

## דרישות לתורמים

בעת כתיבת קוד:
1. אל תשתמש בקריאות API חיצוניות
2. אל תעביר נתונים רגישים
3. בדוק קוד לחולשות נפוצות
4. אל תשתמש בספריות מסוכנות

## יצור קשר

🔒 אבטחה **תמיד בעדיפות**
